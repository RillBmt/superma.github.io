代码随想录刷题总结：

还需要反复做的题目：

T96.不同的二叉搜索树、T416. 分割等和子集、T474. 一和零、T139. 单词拆分、T213. 打家劫舍 II、T337.打家劫舍III、T123. 买卖股票的最佳时机 III、T309. 最佳买卖股票时机含冷冻期、T53. 最大子数组和（dp和贪心两种方法可以再做做）、T115. 不同的子序列

一、动态规划

动态规划做题的关键点：

  1. 对于想不出来的问题，不要只纠结于i和i的前一个状态，有时候还有可能是前j个状态推导出i的状态，即双层for循环
  2. 一维dp数组想不出来的问题，可以考虑考虑用二维dp数组

1.基础题目：

  • 像爬楼梯类的题目，有n级台阶，当n = 0时，算作0级台阶，后面再才是1 2 3…级台阶，所以创建dp数组时大小要设为n + 1；上台阶需要花费时也一样，只不过题目是把开头和结尾元素往后移了，实际上差不多的
  • 像整数划分和不同的二叉搜索树类的题目，并不是靠前面1个或者2个3个这种具体数量的状态来判断当前的状态了，而是根据前面所有的状态来判断当前状态，因此写的一般都是双重for循环，内层从1遍历到当前的i

2.背包问题：

递推公式：

题目问法 递推公式 题目
问能否能装满背包（或者最多装多少） dp[j] = max(dp[j], dp[j - nums[i]] + nums[i]) T416.分割等和子集、T1049.最后一块石头的重量II
问装满背包有几种方法 dp[j] += dp[j - nums[i]] T494.目标和、T518. 零钱兑换 II、T377.组合总和Ⅳ、T70. 爬楼梯进阶版
问背包装满最大价值 dp[j] = max(dp[j], dp[j - weights[i]] + values[i]) T474.一和零
问装满背包所有物品的最小个数 dp[j] = min(dp[j], dp[j - coins[i]] + 1) T322.零钱兑换、T279.完全平方数

遍历顺序：

背包问题 遍历顺序 题目
01背包（二维） 先遍历谁都行，内层正序遍历 01背包二维理论
01背包（一维） 只能先遍历背物体，后遍历背包，且背包倒序遍历（只能取一次） 01背包一维理论
完全背包（求组合数） 先遍历物体，后遍历背包，正序遍历（可取出多次） T518. 零钱兑换 II
完全背包（求排列数） 先遍历背包，后遍历物体，正序遍历（可取出多次） T377.组合总和Ⅳ、T70. 爬楼梯进阶版
完全背包（求最小数） 先遍历谁都行，正序遍历 T322.零钱兑换、T279.完全平方数

  • 01背包

    • 背包问题里面，递推公式往往是dp[j - 数组中的某个值]，所以在内层循环的时候要注意，j 的范围是 > “数组中的某个值”，而不是 > 0

    • 分割等和子集需要进行变通，要求的target = left - right，两部分总和sum = left + right，相加就得到left = (sum + target) / 2，这样就转化成了求left的01背包问题

    • 一和零其实是两个背包的01背包问题，只不过中间还需要记录一下0和1的数量

      • 假如题目改一下，把0全部去掉，只限制1的个数，那么就变成了求只包含1的最大子集长度，就很容易看出来是01背包了

  • 完全背包

    • 根据上面总结的表格来看
    • 单词拆分，要把原wordDict放到set里面，这样才能实现对子串的查找，主要就是找 i 到 j 的子串是否在字典里，如果在而且 i 前面的子串也在，那么j前面的这个长串都能由wordDict构成

3.打家劫舍

  • 打家劫舍II：成环了之后，这里没必要再用一个数组，就用原来的就行，只不过头尾不能都偷之后，就要把头部除去或者把尾部除去，两部分分开考虑，最后取得最大值就行
  • 打家劫舍III：属于一个二叉树上的递归题目，不能再用DP的惯用思想了，依然是遍历，遍历的时候记录节点偷还是不偷两者的值，再把左右的值返回给中间节点，让中间的节点进行偷和不偷的取舍

4.股票问题

  • 这类问题要根据不同的题意，把当天的状态都列举出来，有可能是无买卖状态、当天是买入状态、当天是卖出状态、当天是第一次买入状态、当天是第一次卖出状态、当天是第n次买入或者卖出状态、当天是冷冻期…等等这些状态都要想全；只有把当天的几个状态确定了，后面的递推公式和初始化才好写

5.子序列问题

  • 最长不连续递增子序列和最长连续递增子序列：

    • 不连续递增子序列的跟前0-i 个状态有关
    • 连续递增的子序列只跟前一个状态有关

  • 最长连续公共子数组和最长不连续公共子序列：

    • 连续的子数组只有当前元素相等时和左上角有关系，如果不相等就不用管，因为假如当前值取了左边和上边的最大值，再到右下角元素相等的时候，它把这个值+1，就有错误，因为它不连续，所以就不应该这样操作
    • 对于不连续的公共子序列，不相等时就可以采用上面说的取最大值的方法了，要是相等，还是要把左上角的值拿来再+1

  • 编辑距离问题：

    • 要清楚dp表示的是操作数
    • 如果当前字符相同，那么两个字符串的这个字符就可以忽略，dp就要看左上角的值
    • 如果当前字符不相同，
      • 要么word1删除这个字符，那么dp就是上方的dp值+1
      • 要么word1增加字符，实际就是word2删除字符，那么dp就是左边的dp值+1
      • 要么word1或者word2替换字符，那么dp值就是左上角的值+1

  • 回文子串和回文子序列：

    • 他们都需要一个从前往后的i和从后往前的j，看i到j是否是回文串，或者看i到j中最长的回文串是多少
    • 回文子串：如果当前字母不一样，那肯定不是回文串，就不用管；如果当前字母一样，那么就判断i和j两个下标之间是不是就差了0或者1，或者看i + 1到j - 1的子串是不是回文串；最后统计回文串数目就行
    • 最长回文子序列：如果当前字母一样，那就看i + 1到j - 1的最长回文子串有多长；如果当前字母不一样，就看去掉i或者去掉j后他们之前构成的回文子串有多大，取最大的即可
